# Agentic Travel Planning System

A smart, multi-tool travel planning agent that understands traveller intent, searches options in parallel, builds complete itineraries, resolves conflicts automatically, and handles post-booking changes through a clean dashboard UI.

---

## Assignment Overview

**Assignment:** #14  
**Category:** Agentic AI  
**Group Size:** 5 Students  

### Problem Statement
Travel planning is fragmented and time-consuming. Travellers often spend hours comparing flights, hotels, schedules, and activities manually.  
This project solves that with an **agentic planner** that keeps all constraints in context, executes parallel searches, resolves itinerary conflicts, and adapts to changes.

---

## What This Project Builds

An end-to-end **Agentic Travel Planning System** with:

- Natural-language travel brief understanding
- Parallel flight + accommodation search
- Full itinerary assembly
- Conflict detection and automatic resolution
- Post-booking change management
- Traveller-facing dashboard and chat-based updates

---

## Core Features

### 1) Travel Brief Intake
Parse natural language travel input and extract:

- Origin and destination
- Travel dates
- Number of travellers
- Budget range
- Accommodation preferences
- Special requirements

---

### 2) Agentic Search Execution
Run parallel searches across travel sources to:

- Find flights and stay options
- Compare alternatives
- Filter by user constraints
- Select best-fit combinations

---

### 3) Itinerary Assembly
Generate a complete itinerary including:

- Flights
- Accommodation
- Day-wise activities
- Local transport suggestions
- Restaurant recommendations

All steps are ordered in a logical, time-consistent flow.

---

### 4) Conflict Resolution
Automatically detect and resolve itinerary problems such as:

- Hotel check-in before arrival
- Activity overlap or overruns
- Tight layovers/connections

System applies fixes and explains what changed.

---

### 5) Change Management
Handle real-world disruptions after booking:

- Flight delays
- Cancellations
- Date changes

Agent re-plans affected segments, detects downstream impacts, and proposes valid revised itineraries.

---

### 6) Traveller Dashboard
User-friendly UI with:

- Timeline itinerary view
- Booking confirmations
- Day-by-day schedule
- Maps and travel context
- Chat interface for modification requests

---

## Success Metrics

The system is considered successful when it can:

- Correctly extract all constraints from complex travel briefs
- Return valid flight + accommodation options within constraints
- Produce a logically consistent itinerary with no timing conflicts
- Detect and resolve planted scheduling conflicts
- Handle flight cancellation effects and generate a valid revised itinerary

---

## Why This Is Agentic

Unlike static planners, this system behaves like an agent by:

- Maintaining context across tools and decisions
- Reasoning over constraints and trade-offs
- Taking corrective actions when conflicts arise
- Re-planning dynamically as events change
